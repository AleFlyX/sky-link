# 三天分工文档 - 成员 C（全栈偏前端）

## 负责范围

成员 C 负责前端主流程搭建与核心页面联调，同时承担少量贴近页面的接口适配工作，是三人协作中的联调中枢。

对应前端模块：

- 登录注册页
- 首页框架、路由、导航
- 个人中心
- 通讯录/好友
- 群聊与消息列表
- 文件中心
- 文档列表与编辑页
- 任务页
- 日程页
- 公告通知页

## 落地目标

- 第 1 天结束前，前端框架、登录态和主页骨架可运行。
- 第 2 天结束前，至少 3 个真实联调页面可展示。
- 第 3 天结束前，课程展示优先页面成型，具备完整用户操作路径。

## 任务拆解

### 必做

- 路由和页面框架
- 登录注册页
- 请求封装、Token 管理、错误处理
- 用户/部门/文件/任务/公告主页面
- 基础表格、表单、分页、状态标签组件

### 可延后

- 聊天页的高级交互
- 富文本/Markdown 高级编辑体验
- 非关键页面的精细样式

## Day 1

- 完成前端项目基础搭建：路由、页面骨架、请求封装、Token 存储、统一错误提示。
- 先接入认证和当前用户接口，做登录注册页、基础布局、导航栏。
- 约定前端 API 调用方式、分页组件格式、状态枚举映射。

具体清单：

- 建立 `api`、`views`、`components`、`stores` 基础结构
- 完成登录页、注册页、主布局页
- 完成请求拦截器、401 处理、统一消息提示
- 完成左侧导航或顶部导航骨架

交付物：

- 可运行的前端框架
- 登录后进入主页面的基本流程
- 至少 3 个空页面和统一布局可见

## Day 2

- 联调成员 A：个人中心、用户列表、部门列表、角色展示。
- 联调成员 B：文件列表、任务列表、公告列表。
- 完成前端公共组件：列表页、分页、表单弹窗、状态标签。

具体清单：

- 跑通个人中心、用户管理、部门管理页面
- 跑通文件列表、任务列表、公告列表页面
- 完成表格组件、分页组件、弹窗表单组件
- 统一枚举显示：
  - 用户状态
  - 任务状态
  - 公告类型

交付物：

- 管理页和成员端至少 3 个页面可真实请求数据
- 前端通用组件初版
- 可以录屏展示基础业务流程

## Day 3

- 联调好友、群聊、消息、文档、日程页面。
- 补齐页面交互细节：加载态、空态、错误提示、成功提示。
- 做课程展示优先级最高的页面收尾：登录、用户管理、文件中心、任务管理、公告通知。

具体清单：

- 跑通消息列表或历史消息展示
- 跑通文档列表与详情
- 跑通日程列表与创建
- 对关键页面补搜索、分页、按钮状态、空数据提示
- 统一页面风格，确保演示时不显得割裂

验收标准：

- 从登录到进入系统主界面流程完整
- 至少 5 个核心页面能真实联调展示
- 页面样式统一，接口异常有基本反馈
- 演示时可以从“登录 -> 管理/成员业务 -> 结果展示”完整走一遍

## 提交节奏

- Day 1 上午搭框架，晚上完成登录和主布局
- Day 2 下午前完成列表类页面第一版
- Day 2 晚上开始集中联调
- Day 3 以收口、修样式、修交互为主，不再大改架构

## 风险与兜底

- 如果后端接口未完成，先用 mock 数据把页面结构和交互补齐。
- 如果聊天页联调困难，至少保证会话列表和历史消息静态展示。
- 如果文档编辑器来不及，先做文档列表、详情和普通文本编辑。

## 协作说明

- 成员 C 不需要先做很重的后端业务，只需要补贴近页面的小接口适配或 VO 转换。
- Day 1 优先等成员 A 的登录接口；Day 2 开始并行接 A、B 两条线。
- 如果后端未完成，先用 mock 数据占位，保证页面结构和交互先成型。
